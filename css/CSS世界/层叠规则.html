    # 层叠规则

    层叠上下文指的是具有层叠关系的作用域，其可以嵌套，就像JS中函数划定了一个作用域，处于同一作用域的的同级别元素，DOM位置处于后面的就会覆盖前面的， 不在同一作用域的元素层级就由其具有层叠上下文的父级或祖先元素决定，元素的层叠上下文级别始终受限于其父级或祖先元素的层叠上下文级别（包括z-index为负值也无法超出所在层叠上下文）。
    层叠水平(或顺序、层级)指元素在界面文档中Z轴方向上的高低级别，每个元素都具有层叠水平，只是一般都默认是按文档流显示(注意其与z-index:auto并不相同)，只有部分属性可以影响或改变层叠顺序高低，如我们熟知的z-index，但也并不是只有该属性可以影响。

    • z-index属性只有和position不为static的定位元素(或是flex盒子的子元素)在一起的时候才有作用，可以是正数也可以是负数。
    • 注意z-index值默认是auto不是0，单纯从层叠顺序上来看可以看作auto和0同级，但含义并不一样，auto并不会触发层叠上下文，而0会。

    # 层叠上下文的创建

    • 根元素，即html元素，其本身具有层叠上下文———毕竟页面所有内容都在其内部。
    • 对于position值为relative/absolute以及Firefox/IE(不包括Chrome、Safari等webkit浏览器)下含有position:fixed声明的定位元素，当其z-index值不是auto的时候，会创建层叠上下文。 Chrome、Safari等webkit内核浏览器中含有position:fixed声明的定位元素自动具有层叠上下文，无须设置z-index为数值来触发。 也就是说，只设置了position时并不表示其就具有了层叠上下文，若想具有层叠上下文，又不像改变其层叠级别，则就可以将z-index设置为0。
    • 除我们常见的定位加z-index可以创建层叠上下文，由于CSS3的出现也使得其他一些属性可以创建层叠上下文，这部分的层叠顺序与z-index为auto或0处在同一级别。
      • 元素为flex布局元素（父元素display:flex|inlineflex），同时z-index值不是auto。
      • 元素的opacity值不是1。
      • 元素的transform值不是none。
      • 元素mix-blend-mode值不是normal。
      • 元素的filter值不是none。
      • 元素的isolation值是isolate。
      • 元素的will-change属性值为上面2～6项的任意一个（如will-change:opacity、will-chang:transform等）。
      • 元素的-webkit-overflow-scrolling设为touch。

    # 层叠上下文与层叠顺序的关系

    • 若层叠上下文元素不依赖z-index数值，则其层叠顺序是z-index:auto，可看成z-index:0级别；
    • 若层叠上下文元素要依赖z-index数值，则其层叠顺序由z-index的数值大小决定。
