# 方法覆写时使用原始值

在覆写已存在对象的某属性或方法又要用到该属性或方法的原始值时，一种方法是提前将原始值单独用个变量存起来以备覆写时内部使用，另一种无需新增额外变量的方法是使用返回新内容的立即执行函数将原始值作为参数传入，从而在函数内容使用传入的原始值：

// 假设 obj.foo 已存在初始值
obj.foo = (function(originFoo) {
  originFoo();  // 原始值，有时可能需要这样调用：originFoo(this)

  return function() {
    console.log('new method');
  }
})(obj.foo);

# 监听DOM节点变化

可使用MutationObserver构造方法或MutationEvent相关事件(最新标准中已删除)监听DOM节点变化，包括元素的添加/移除、属性添加/修改(包括行内style)、文本修改，以及子节点的任何修改都可监听到。可用于添加水印等场景，一定程度上避免在浏览器调试模式下删除或修改style隐藏水印，这里说一定程度上是因为无法完全避免通过修改样式的方式来隐藏水印，如果我们修改的不是html标签上的直接内容，而只是最终作用在其上的CSS，此时是不会触发变化监听的，因为CSS只是逻辑上应用在元素上，而并未直接改变DOM节点上的内容，比如元素上本就有class，我们通过class修改样式也就不会改动标签上的任何东西，即使没有class、id等也依然可以通过其他特征添加选择器将CSS应用在当前元素上，而不修改元素标签上任何内容。

# .nvmrc文件指定

使用 nvm 管理 node 版本的可在项目根目录使用 .nvmrc 文件指定当前工程应用的 node 版本，内容示例如下：

v14.19.1

创建 .nvmrc 文件并写入版本（或更新版本）：

# 将当前版本写入
node -v > .nvmrc

# 指定使用 5.9 的版本
echo "5.9" > .nvmrc

# 指定使用最新的 LTS （Long-term Support） 版本
$ echo "lts/*" > .nvmrc

使用 nvm use 命令时就会默认根据 .nvmrc 文件中的版本号自动切换到对应的版本，若本地没有对应的版本则使用 nvm install 即可默认安装相应版本。

# package.json文件指定

通过 package.json 文件中 engines 字段来指定版本范围。

指定node版本的范围：

{
  "engines": {
    "node": ">=14.19.1 <=17.9.0"
  }
}

固定为某版本：

{
  "engines": {
    "node": "~14.19.1"
  }
}
