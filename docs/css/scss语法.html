# SCSS基础语法

SCSS编译输出css文件内容格式有四种：nested包含层级缩进、expanded无层级缩进但每个属性占一行、compact同作用域的属性放在一行、compressed完全压缩。

  • &符号表示当前位置的上一层
  • @at-root表示跳出当前嵌套层级到最外层

SCSS具有运算的特性，可以对数值型的value(如数字、颜色、变量等)进行加减乘除四则运算（其中一个计算数可带单位），注意运算符前后请留一个空格，不然会出错。

SCSS中使用==和!=进行等值比较。

# 变量声明

  • 变量名定义已$开头：$asideWidth: 200px
  • 变量值后的!default表示默认值，有该修饰符时，其他与该变量同名的声明不管是在当前位置之前还是之后，都可以覆盖当前默认值，否则只有在后面声明的可以覆盖前面的同名变量值
  • 在最外部声明的则是全局变量，在函数或某嵌套层声明的就是局部变量，与JS中的变量作用域类似
  • 局部变量后加上!global 修饰符则表示全局变量，将覆盖最外层的同名变量，但在未声明前访问的还是未覆盖的原变量值（类似JS变量未二次赋值前）

一般定义的变量都为属性值，可直接使用，但当变量作为属性名或字符串拼接等特殊情况下使用时，需要以#{$var}插值的形式使用。

  • 拼接字符串：#{$pd}-left: 10px
  • CSS中的calc()计算方法中就要使用该插值方式：width: calc(100% - #{$asideWidth})
  • CSS中以/分隔的属性值：font: #{$fz}/#{$lh} fontFamily，此处若不加#{}，分隔符/就会被当做除号，编译出错误的结果
  • 将 scss 变量用在设置原生 css 变量的值时也要使用该插值方式：--e-primary: #{$primary}

# List与Map数据

为方便我将其称为组值变量，指的是变量值有多个，可以将其整体当做一个值(就像CSS中的一些属性合并写法的值)，也可以当做是一组值(像数组或对象)。

组值分为List类型和Map类型，简单来说List类型有点像JS中的数组，而Map类型有点像JS中的对象(或ES6的Map)

  • List数据可通过空格、逗号、小括号分隔多个值（JS中数组只能是逗号分隔），且可省略最外层小括号
    • 可用nth($var, $index)方法取组值变量$var中$index索引位置的值，注意索引从1开始，而非0
    • 关于List数据操作还有很多其他函数，需要的时候可查阅文档补充
  • Map数据类似对象以key: value键值对形式构成，只是最外层不是用花括号包裹，而是用小括号(不可省略)，其中value也可以是组值
    • 格式：$map: (key1: value1, key2: value2, key3: value3)，其键值对的访问顺序与List(或数组)内容一样是按声明时顺序来的（这与JS中的对象不同）
    • 可用map-get($map, $key)方法读取变量$map中$key键的值
    • 实际上Map数据是List数据的一种特殊形式，Map数据被当做List使用时会自动按key value为一个单元格值的形式转换为二维List：(k1: v1, k2: v2)会被转换为(k1 v1, k2 v2)
    • 由于Map就是特殊的List，所以List可用的方法Map也都可以用
    • 关于Map数据操作还有很多其他函数，需要的时候可查阅文档补充
$list1-1: 5px 10px 20px 30px;    // 一维数据，在某些CSS属性中也可当做一个整体值
$list1-2: 5px, 10px, 20px, 30px;  // 一维数据，同上面 $list1-1，但若用于的CSS属性值格式没有逗号分隔这里却有逗号时则不能当做一个整体值
$list2-1: 5px 10px, 20px 30px;    // 二维数据，相当于JS中的二维数组
$list2-2: (5px 10px) (20px 30px);   // 二维数据，同上面 $list2-1 作用一样
$list2-3: (12 12px) (14 14px);   // 二维数据

$map: (h1: 20px, h2: 18px, h3: 16px);  // Map数据

a {
  font-size: nth($list1, 3);    // 下标从 1 开始

  &:hover {
    // 访问多维数据，由内向外一层层嵌套查找
    font-size: nth(nth($list2-1, 2), 1);   // 这里表示取出 $list2-1 中第 2 组值后，再从中取出第 1 个值
  }
}

# 函数声明

  • @function关键字声明函数
  • @return关键字返回内容
  • 函数参数也可像ES6中一样指定默认值
@function foo($arg){
  @return ($arg / 75) * 1rem;
}
// 给参数指定默认值（注意与ES6中不一样，ES6中参数指定默认值使用“=”，而“:”是TS中声明类型）
@function foo($arg: 30){
  @return ($arg / 75) * 1rem;
}

# 遍历与循环

  • @each $item in <list or map>用于遍历组值(List和Map数据)，类似JS中的for...in
    • $item组值中的每一项值，若每一项有多个值，也可以类似ES6中的解构赋值进行展开：@each $item1-1,$item1-2, in $list2
  • for循环有两种形式，分别为：@for $i from <start> through <end>和@for $i from <start> to <end>。
    • $i表示从start到end的逐一递增整数值，start表示起始值，end表示结束值，这两个for循环的区别是关键字through会包括end这个索引，而to则不包括end这个索引
  • @while循环@while <expression> { ... }，表达式expression若为真则执行循环体内容直到其值为假才结束循环，可在循环体中修改表达式expression中用到的变量值使得达到某条件结束循环
// 由于`Map`就是特殊的`List`，故遍历`List`的方式也都可以用于遍历`Map`
// List数据
$sizeList: (0 0)  (5 5px)  (10 10px) (15 15px) (25 25px) (50 50px) !default;
$classList: (pd padding) (pdt padding-top) (pdr padding-right) (pdb padding-bottom) (pdl padding-left) (mg margin) (mgt margin-top) (mgr margin-right) (mgl margin-left) (mgb margin-bottom) !default;
// Map数据
$h-fz: (h1: 20px, h2: 18px, h3: 16px);

// @each...in 遍历
// 方式一
@each $className in $classList {  // 取出单元格每一项
  // 局部变量
  $name: nth($className, 1);  // 表示取 $className 中的第 1 个值（索引从1开始）
  $attr: nth($className, 2);

  @each $val in $sizeList {
    $cname: nth($val, 1);
    $cval: nth($val, 2);
    .#{$name}-#{$cname} {
      #{$attr}: #{$cval} !important;
    }
  }
}
@each $hz in $h-fz {  // 此时取出的单元格值结构为：(key value) 这样的一维 List
  #{nth($hz, 1)} {
    font-size: map-get($h-fz, nth($hz, 1));
  }
}
@each $hz in $h-fz {  // 将 $h-fz 当做 List 使用
  #{nth($hz, 1)} {
    font-size: nth($hz, 2);
  }
}

// 方式二
@each $name1, $attr in $classList {  // 取出单元格每一项并展开，按顺序对应单元格中的数据（类似解构赋值）
  @each $name2, $val in $sizeList {
    .#{$name1}-#{$name2} {
      #{$attr}: #{$val} !important;
    }
  }
}
@each $h, $fz in $h-fz {  // Map数据展开则分别对应 key、value
  #{$h} {
    font-size: $fz;
  }
}

// @for...through | to 循环
@for $i from 1 through 3 {  // $i 依次为 1、2、3
  .item-#{$i} { width: 2em * $i; }
}
@for $i from 1 to 3 {  // $i 依次为 1、2，不包含 3
  .item-#{$i} { width: 2em * $i; }
}

# 条件判断

  • 流控条件判断由关键字@if、@else if和@else组成
  • 三目判断方法if($condition, $if_true, $if_false)，类似JS中的? :，三个参数分别表示：条件$condition，条件为真的值$if_true，条件为假的值$if_false
$dir: left;

div {
  font-size: if($dir == left, 12px, 14px);

  @if $dir == left {
    border-width: 5px 10px 5px 0;
  }
  @else if $dir == right {
    border-width: 5px 0 5px 10px;
  }
  @else if $dir == top {
    border-width: 0 5px 10px 5px;
  }
  @else {
    border-width: 10px 5px 0 5px;
  }
}

# 混入@mixin

混入就是将定义的内容混合进(插入)引用它的位置。

  • 通过@mixin mixinName { 内容 }声明混入内容，可以将其看作是一个函数，返回值就是混入的内容，混入名也可以类似函数传入参数@mixin mixinName($args) { 内容 }，参数可像函数一样指定默认值
  • 通过@include mixinName或@include mixinName($args)来引用对应的混入内容
  • 在传入参数时，若参数有多个值(或不确定几个值)，而内部又只是将所有传入的参数当做一个值时，可在变量后加三个点...，这样当传入多个用逗号分隔的值时不会被当做是不同部分参数
    • @include mixinName($args...)这样在内容中$args变量将包含传入的所有值（主要是一些CSS属性可用逗号分开设置多个值的情况，如背景图、阴影等），注意与ES6中的扩展运算符...进行区分(一个在前，一个在后)
  • 通过@content关键字指代引用某 mixin 时传入的自定义内容(类似slot)，这样在声明 mixin 时其内容可部分或完全由外部自定义（注意不同于引用 mixin 时传入的参数 $args）
//定义一个“清除浮动”的 mixin，混入名为“clearfix”
@mixin clearfix {
  &:after {
    content: "";
    display: table;
    clear: both;
  }
}

// 当做函数传入参数
// 设置浏览前缀，实际相关工具配置 autoprefix 可自动完成
@mixin prefix($property, $value) {
  // 不需要设置return，自动返回所有内容
  @each $prefix in -webkit-, -moz-, -ms-, -o-, '' {
    #{$prefix}#{$property}: $value;
  }
}
// 在变量后加三个点...可省略后面的形参声明，此时传入的所有参数(指逗号隔开的情况，空格隔开只算是一个组合值)都当做一个值由第一个参数接收
@mixin boxShadow($shadow...) {
  box-shadow: $shadow;
}

// @content 自定义内容
// 定义动画
@mixin keyframes($name) {
  @-webkit-keyframes #{$name} {
    @content;  // @content 接收外部一整块样式
  }
  @-moz-keyframes #{$name} {
    @content;
  }
  @keyframes #{$name} {
    @content;
  }
}

// @include 引用相关 mixin

// 外部引用
@include keyframes(move) {
  // 这里面的内容都将被 keyframes mixin 中的 @content 接收
  0% {
    transform: translateX(0);
  }
  100% {
    transform: translateX(100%);
  }
}
// 内部引用
.e-row {
  @include clearfix;
  @include boxShadow(0 0 10px 1px #eee, 0 2px 2px #666, 0 4px 4px #999);

  .e-col {
    float: left;
    animation: move 0.3s forwards;
  }
}

# 继承

可以通过@extend selector语法让当前选择器继承selector选择器的所有样式，selector可以是CSS中明确的选择器，也可以是SCSS定义的占位选择器

  • selector为CSS常规选择器时，编译后CSS中会在继承的选择器后并列一个当前选择器，从而共用样式
  • selector为SCSS占位选择器时，编译后CSS中会将占位选择器的内容复制一份到当前选择器下，有点类似@mixin的作用
    • 占位选择器声明以%开头：%selector { 内容 }
h1{
  border: 4px solid #ff9aa9;
}
.speaker{
  @extend h1;   //继承h1的样式
  border-width: 2px;
}
// 以上编译后CSS结果为
h1,
.speaker {
  border: 4px solid #ff9aa9;
}
.speaker {
  border-width: 2px;
}

//占位选择器 %，有点类似 @mixin 的作用
%placeholder{
  border-radius: 5px;
}
div {
  @extend %placeholder;    //若此处不用@extend调用，则上面定义的内容就不会编译在css文件中
}
// 以上编译后CSS结果为
div {
  border-radius: 5px;
}

# 颜色处理

列举几个常用颜色处理函数。

  • lighten($color, $amount)减淡，参数$color为基准颜色值，$amount为百分比表示加深或减淡的相对程度，注意这并不是增加透明度
  • darken($color, $amount)加深
  • rgba($color, $alpha)增加透明度，该方法返回颜色$color指定透明度$alpha后的rgba格式色值，若传入四个参数就与CSS原始rgba没什么区别了
  • mix($color1, $color2, [$weight])，该方法返回(任意有效色值)颜色$color1按$weight比例与颜色$color2(剩余比例)混合后的色值，默认各50%比例混合
  • grayscale($color)，返回一个与颜色$color有相同亮度的灰度颜色
  • hue($color)，返回颜色$color在色轮上的角度（0deg-360deg）
  • invert($color, $weight)，返回颜色$color的反转颜色
.lightgray{
  color: lighten($gray, 10%);
}
.darkgray{
  color: darken($gray, 10%);
}
.rgba{
  color: rgba(#c82858, 0.65); // rgba(200, 40, 88, 0.65)，根据`#c82858`的`65%`透明度计算出一个rgba颜色值
}
.mix{
  color: mix(#c82858, rgba(200,40,80,.65), 0.3); //rgba(200, 40, 80, 0.65105)，把`#c82858`和`rgba(200,40,88,.65)`两颜色按`0.3`和`0.7`的比例混合得到一个新颜色
}

# 常用scss

// 清除浮动
.clearfix{
  &:after,
  &:before {
    content: '';
    display: table;
    height: 0;
  }
  &:after {
    clear: both;
  }
}

// px 转 rem（此处以iPhone6）
@function px2rem($int){
  @return ($int / 75) * 1rem;
}

// 设置webkit浏览器中的滚动条样式
@mixin scrollBar {
  &::-webkit-scrollbar-track-piece {
    background: #d3dce6;
  }
  &::-webkit-scrollbar {
    width: 6px;
  }
  &::-webkit-scrollbar-thumb {
    background: #99a9bf;
    border-radius: 20px;
  }
}

// 设置字体大小（结合flexable.js使用时html上有设置data-dpr属性）
@mixin fz($fz){
  font-size: $fz * 1px;
  [data-dpr="2"] & { font-size: $fz * 2px; }  // 这里的 & 指代所在选择器层级，此处会被提取到最外层，并不会在引入的层级下再嵌套 [data-dpr="x"]
  [data-dpr="3"] & { font-size: $fz * 3px; }
}

// 多行文本溢出省略
@mixin ellipsis($num:1) {
  display: -webkit-box;
  -webkit-box-orient: vertical;
  -webkit-line-clamp: $num;
  overflow: hidden;
  text-overflow: ellipsis;
  text-align: justify;
  word-break: break-all;
}

// 设置阴影
@mixin boxShadow($x, $y, $color) {
  -moz-box-shadow: $x $y 4px $color; // FF3.5+
  -webkit-box-shadow: $x $y 4px $color; // Saf3.0+, Chrome
  box-shadow: $x $y 4px $color; // Opera 10.5, IE 9.0
  -ms-filter: "progid:DXImageTransform.Microsoft.dropshadow(OffX=#{$x}, OffY=#{$y}, Color='#{$color}')";  // IE8
  filter: progid:DXImageTransform.Microsoft.dropshadow(OffX=#{$x}, OffY=#{$y}, Color='#{$color}');    // IE6,IE7
}

//渐变色
@mixin gradient2right($startColor, $endColor) {
  background-image: -webkit-gradient(linear, left bottom, right top, color-stop(0%, $startColor), color-stop(100%, $endColor));
  background-image: -webkit-linear-gradient(left, $startColor, $endColor);
  background-image: -o-linear-gradient(left, $startColor, $endColor);
  background-image: -moz-linear-gradient(left, $startColor, $endColor);
  background-image: -ms-linear-gradient(left, $startColor, $endColor);
  background-image: linear-gradient(to right, $startColor, $endColor);
}

// 绘制三角形
@mixin triangle($size, $color, $direction) {
  $halfSize: $size/2;
  border: solid transparent;

  @if $direction == up {
    border-width: 0 $halfSize $size $halfSize;
    border-bottom-color: $color;
  }

  @else if $direction == right {
    border-width: $halfSize 0 $halfSize $size;
    border-left-color: $color;
  }

  @else if $direction == down {
    border-width: $size $halfSize 0 $halfSize;
    border-top-color: $color;
  }

  @else if $direction == left {
    border-width: $halfSize $size $halfSize 0;
    border-right-color: $color;
  }
}
