# 第 6 章 全新的布局方式

# 多列布局

多列布局最初的目的是用于解决文本排版，像一些期刊杂志、报纸内容的排版（也可很方便地实现图片的瀑布流布局）。demo 示例

# columns

多列布局大多场景只需要用到columns属性即可，有时也会用到columns-gap设置列间间隔距离。

columns属性是columns-count和columns-width属性的缩写，且值的顺序无关，会自动根据值的类型匹配属于哪个属性。columns: <count> <width>;或columns: <width> <count>;均可，且两个值都是可以选的，默认都为auto。

columns-count表示想要分成的“理想”的列数，值为正整数或auto(默认值)。

columns-width表示每列的“最佳”宽度，值为表示尺寸的长度值(不支持百分比值)或auto(默认值)。

上面说的“理想”和“最佳”的意思就是浏览器只是尽量往设定的值靠拢，而并不一定就刚好是这个值，最终实际多少列是由columns-count和columns-width这两个属性共同决定的，并没有哪个属性的优先级一定更高，最终实际列数是按照最小原则决定的，即看columns-count设定的列数与将整行按columns-width宽度可分割出的列数哪个值更小就是哪个。

多列布局的每一列的高度并不总是相等的，内容的分隔也不总是均匀的，浏览器有自己的一套算法。

# columns-gap/gap

columns-gap表示每一列之间的空白间隙大小，可以是长度值，也可以是百分比值(相对整行)。

也可直接使用gap属性替代(作用一样)：gap属性实际是columns-gap属性和row-gap属性的缩写，是统一多列布局、Flex 弹性布局和 Grid 网格布局的间隙设置的属性。

# columns-rule

columns-rule用于设置每列之间的分割线样式，语法同border和outline，且不占据空间，其也是columns-rule-style/columns-rule-width/columns-rule-color三个属性的简写。

# columns-span

columns-span类似 table 表格中的solspan列合并(html 属性，非 css 属性，表格另有rowspan行合并，但这里没有)，该属性设置在多列布局的子元素上，表格该内容是否跨多列展示。

其只有两个值：

  • none，表示不跨列展示，默认值
  • all，表示跨列展示，且是跨所有列，即占用整行

# columns-fill

columns-fill用于设置如何去分配内容填充在每一列中（一般容器有设置高度才容易看出各值的差异），默认balance，有如下几个值：

  • balance，表示尽可能平衡各列的内容量，使每列的内容量差不多
  • auto，表示按顺序填充每一列，前一列内容满了再换到第二列，以此类推
  • balance-all，表示尽可能平衡各列的内容量，使每列的内容量差不多，即使是分隔断开的上下文，目前无浏览器支持无法查看效果。

# 两端对齐和元素断点位置控制

多列布局的首尾列会自动两端对齐，再设置各列间隙gap即可，唯一缺点是只适合实现单行内容的两端对齐效果（这里单行只是的从容器整体来看，不存在行数分割）。

break-inside属性(并非多列布局专用)可定义页面打印、多列布局中内容发生中断(即本该是放在同一块中的内容被分割在了多个列中，比如同一个段落的内容可能第一列还有空间填充了一部分，剩余的又在第二列)时元素的表现形式。该属性有几个可选值：auto、avoid、avoid-page、avoid-column等。

  • auto，表示元素可以中断
  • avoid，表示元素不能中断，若我们希望每个列元素都是独立的，不能被拆分到多个列中，可以设置该值

默认情况下，元素片段在跨行/列/页(如打印)时的样式是分隔渲染的，如边框或者圆角都是分割开的(就像直接切开的)，若希望即使元素内容因为分列断开，其一些像边框/阴影等属性能完整渲染而不是被分割，则可以使用box-decoration-break属性实现，其有两个值：

  • slice，即断开，如同被切开一样，为默认值
  • clone，表示断开位置的各元素样式独自渲染，就像被复制了一份，而不是直接切开

box-decoration-break属性并非对所有样式都有效，其只能影响部分 CSS 属性的渲染，有效的包含：background、border、border-image、border-radius、box-shadow、margin、padding、clip-path。但是该属性目前在 Chrome 和 Safari 下只是部分支持，对多列和分页场景暂不支持，而像普通内联元素换行的情况是支持的。

# Flex 弹性布局

demo 及属性示例

flex: none | auto | [ <flex-grow> <flex-shrink>? || <flex-basis> ]；flex-grow表示有多余空间时对多余空间的分配比例，默认值0；flex-shrink表示空间不够时需要压缩的空间分配比例，默认值1； flex-basis表示元素的基本固定宽度，默认值auto(但 Chrome 较新版本中默认是0)，必须是一个表示尺寸的值。

  • flex: none，此时同flex: 0 0 auto，即flex-grow: 0、flex-shrink: 0、flex-basis: auto；
  • flex: auto，此时同flex: 1 1 auto，即flex-grow: 1、flex-shrink: 1、flex-basis: auto；
  • flex: 2，只有一个数值时，该值表示flex-grow，其余两个值使用默认值，也即等同于flex: 2 1 auto，但 Chrome 较新版本中等同于flex: 2 1 0；
  • flex: 100px，只有一个尺寸值时，该值表示flex-basis，其余两个值使用默认值，也即等同于flex: 0 1 100px，但 Chrome 较新版本中等同于flex: 1 1 100px；
  • flex: 2 2，两个数值时，该值表示flex-grow flex-shrink，而flex-basis使用默认值auto，也即等同于flex: 2 2 auto，但 Chrome 较新版本中等同于flex: 2 2 0；
  • flex: 2 100px，两个值且第二个为尺寸值时，该值表示flex-grow flex-basis，而flex-shrink使用默认值1，也即等同于flex: 2 1 100px；
  • flex: 1 2 100px，三个值时，该值就表示flex-grow flex-shrink flex-basis。

# Grid 网格布局

参考文章 (opens new window)

display: grid;为块级网格，display: inline-grid;为行内块网格，若不设置其他属性则看不出他们内部子元素的排列特殊性，默认就是与display: block;一样的瀑布式自上而下布局 。

# grid-template-columns 和 grid-template-rows

这两个属性用于指定网格的行列数量和尺寸等信息，以下介绍grid-template-columns属性的相关特性和值对grid-template-rows都适用。网格尺寸中有个特殊的单位 fr，其为网格剩余空间的分配比例单位，有点类似 flex 布局中flex-grow属性值的效果（对剩余或多余空间的分配比例）。

/* 这里每个数值就表示对应列的宽度，而值的个数就表示列数多少，这里有3个值就表示3列，并非直接指定一个数字来表示几列 */
grid-template-columns: 200px auto 20%;

grid-template-rows: 5vw 80px auto 20%; /* 该属性对应行的设置，与列类似 */

/* 简写属性 grid-template */
grid-template: rows / columns; /* 该属性还包含 grid-template-areas 属性，是这3个属性的缩写 */
/* 更简写属性 grid，该属性是包含更多其他属性的缩写 */
grid: rows / columns;

# 网格线命名

可以给网格线命名，注意是网格线，不是网格的行或列，以grid-template-columns列为例，语法如下：

/* 这里网格线名使用中括号[]并非可选的意思，而是值需要使用中括号[]包裹，不过确实线条命名也是可省略的，不需要的区域可以省略 */
/* 由于网格中间区域的线是两边格子公用的，所以起名字时也可以起两个以分开使用，但需要放在同一个[]中用空格隔开（一个值即相当于两个同名） */
grid-template-columns: [线名-开始] 尺寸1 [线名2] 尺寸2 [线名3-左 线名3-右] 尺寸3 [线名-结束];

网格线命名主要用于grid-column-start、grid-column-end、grid-row-start、grid-row-end等属性，该功能作用在 grid 子元素上，方便描述 grid 子元素占据的网格区域，一般在语义明确的页面级布局才可能用到，而其他情况几乎用不到。
