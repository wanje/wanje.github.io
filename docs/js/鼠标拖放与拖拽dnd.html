# 鼠标拖拽与拖放

  • 鼠标拖放
  • 原生拖放事件

这里分两种情况：

  1. 鼠标拖拽，这里强调鼠标主要指由鼠标事件组合的常规拖动与停下操作，通常由mousedown/touchstart/pointerdown、mousemove/touchmove/pointermove、mouseup/touchend/pointerup等组成；
  2. 拖放，主要指JS中原生支持的drag拖动与drop放置相关事件构成的操作，通常由drag、dragstart、dragend、dragenter、dragleave、dragover、drop等组成，在原生不支持拖放事件时也是可由上面的鼠标事件模拟实现的。

对于放操作

  • 常规的拖动位置可能最后只是停下拖动，除了位置移动可能并不需要额外操作（当然也可以进行额外操作，只是此时大多情况下可能没有drop方便）
  • 应用原生drop放置事件的情况，一般都有额外的操作，而不只是位置变化，即使位置变化大多也包含对象自身(或克隆对象或映射的对象)html结构上的位置同步变化，而不只是对象坐标移动，如内容挪动、文件拖放上传等

以上两种都是拖拽或拖放，也叫drag-and-drop，简称dnd，只是使用场景不同，可能分开使用，也可能结合使用。

# 鼠标拖放

鼠标拖放算法的基础流程大致如下（可用事件委托处理）：

  1. 在mousedown时，根据需要准备要移动的元素（可能是事件源本身，也可能是创建的一个事件源副本，还可能是映射的其他任何东西）；
  2. 然后在mousemove时，通过更改position:absolute等情况下的left/top来移动它（或使用translate），此时通常将事件绑定在document或body等表示元素可拖动范围的容器上，而不是移动目标上，以避免鼠标与移动目标脱离；
  3. 在mouseup时，执行停下或放置预期的拖放结果相关的所有行为。

关于拖动过程中的坐标应用，为方便坐标计算，有个很好的思路是：拖动用fixed放置再转absolute，若拖动过程中可放置的范围超出可视区(容器内部可以滚动时)，无需每次在拖动mousemove事件中去进行坐标计算，而是先使用fixed固定定位(此时直接用clientX/clientY就很方便)，在最后放置时mouseup事件中再换成absolute绝对定位去计算一次坐标即可。若希望拖动到边缘时容器可滚动(要求容器滚动高度大于其视口高度)以允许进一步的拖动，可通过比较坐标位置与容器边缘位置，再用scrollBy方法逐步滚动容器。

以拖动一个元素到另一个元素上方时，高亮显示目标位置元素的效果为例：

// 抓取欲拖动对象，事件绑定可用 addEventListener 替换（方便使用事件代理）
el.onmousedown = function(e) {
  // 记录鼠标在元素上的抓取位置相对元素自身的偏移，clientX/clientY是相对视窗的坐标
  const elBox = el.getBoundingClientRect(); // 该方法得到的结果中坐标是相对视窗的
  let shiftX = e.clientX - elBox.left;
  let shiftY = e.clientY - elBox.top;

  // (1) 准备移动：确保 absolute，并通过设置 z-index 以确元素在顶部（项目中CSS预设好也可以）
  el.style.position = 'absolute';
  el.style.zIndex = 1000;

  // 将目标元素剪切到限制其拖动范围的视窗容器元素中（只要视觉无影响，当然也可以不做这一步）
  const viewEl = document.body; // 这里拖动范围为整个 body（可根据实际情况来决定）
  viewEl.append(el);
  const viewport = viewEl.getBoundingClientRect();
  const maxLeft = viewport.width - elBox.width;
  const maxTop = viewport.height - elBox.height;

  // 将元素移动到鼠标位置 (clientX, clientY) 坐标上(这里传入的坐标用的鼠标相对整个页面视窗的坐标)
  function moveAt(clientX, clientY) {
    // 这里考虑了拖动范围限制容器的位置
    const deltaX = clientX - viewport.left;
    const deltaY = clientY - viewport.top;
    // 这里考虑了鼠标相对元素自身的位置偏移量，故不会出现按下鼠标时元素可能位置跳动的现象
    let left = Math.max(0, deltaX - shiftX);
    let top = Math.max(0, deltaY - shiftY);
    // 这里考虑了可拖动范围边界限制
    left = Math.min(maxLeft, left);
    top = Math.min(maxTop, top);

    el.style.left = left + 'px';
    el.style.top = top + 'px';
  }

  // 对于初始按自然流式布局的元素可调用一次设置初始定位位置，而对于初始就设置了定位位置的情况此处可以不再调用设置初始位置
  moveAt(e.clientX, e.clientY);

  // 高亮拖动过程中当前覆盖到的元素
  function highlightAt(clientX, clientY) {
    el.hidden = true; // 隐藏拖动的元素（因其始终在最上层，不隐藏则用elementFromPoint定位到的元素始终是拖动对象）

    // elementFromPoint方法返回文档中指定坐标位置(视窗坐标，而非文档或屏幕坐标)的最上层元素（视觉上Z轴的最上层，而非html结构上的最外层）
    // 另有 elementsFromPoint 方法是返回对应位置上的一组元素（包含了对应位置Z轴上的所有层级的元素，包括嵌套的元素[类似事件冒泡经过的所有元素]）
    const highlightEl = document.elementFromPoint(clientX, clientY);

    el.hidden = false;  // 获取完元素就复原显示，整个过程代码执行耗时很短，UI还未更新，故视觉上不会出现闪烁

    if (!highlightEl) return; // 若所在坐标没有元素则不继续处理（坐标在窗口之外时为null）

    // 如果是目标放置对象，则做相关操作
    highlightEl.style.outline = '1px solid lightblue';
  }

  function handleMouseMove(e) {
    moveAt(e.clientX, e.clientY); // 移动的坐标位置需要根据可拖动范围来决定
    highlightAt(e.clientX, e.clientY);
  }

  // (2) 在 mousemove 事件中移动元素，将 move 事件绑定在外层容器上避免鼠标脱钩
  //! 此处需要注意，move事件处理函数可能并不适合进行节流处理，尤其是其中的操作会影响up事件中最终操作的时候，因为节流有可能会导致最后一次的move事件处理发生在up事件处理之后
  document.addEventListener('mousemove', handleMouseMove);

  // (3) 放下拖动对象，并移除不需要的事件处理程序
  el.onmouseup = function() {
    document.removeEventListener('mousemove', handleMouseMove);
    el.onmouseup = null;
  };

};
// 对于图片等特殊元素，浏览器有特殊的拖放处理，此时就会存在冲突（原生拖拽事件影响导致不会触发mouseup）
// 故需要阻止默认的拖放事件（常规布局元素无需特殊处理）
el.ondragstart = function() {
  return false;
};
