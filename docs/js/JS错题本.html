# 易错结果

# parseInt(0.0000005) -> 5

parseInt(0.000005); // -> 0
parseInt(0.0000005); // -> 5
parseInt('0.0000005'); // -> 0

parseFloat('5e-7xde'); // -> 5e-7，也即是 0.0000005

因为 parseInt 是将字符串转为整数，当参数不是字符串时会先被隐式转为字符串（parseFloat 也一样，但其能识别科学计数法形式的字符串）再转为整数。而这种绝对值小于1且小数位前面有至少 6 个 0 的小数（即至少第 7 位开始才是非 0）会用科学计数法表示，转为字符串的时候自然也是科学计数法的形式。

parseInt(0.0000005) -> parseInt(String(0.0000005)) -> parseInt('5e-7') -> 5

# [] == ![] -> true

这涉及到JS中值比较时的隐式转换（除全等比较）：进行比较的值不是同类型时，会先尝试转换为数字再比较

[] == ![] -> [] == false -> '' == 0 -> 0 == 0 -> true

# try...finally

(() => {
  try {
    return 1;
  } finally {
    // 因为 finally 始终会执行，所以这里的 return 覆盖了前面的 return
    return 2; 
  }
})(); // -> 2

在 JS 函数中，try...catch...finally 语句中 finally 代码块始终会执行，无论 try 或 catch 代码块中的代码是否执行或抛出了错误。
