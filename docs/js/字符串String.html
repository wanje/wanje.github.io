# 字符串String

文章：JavaScript字符串操作方法大全，包含ES6方法 (opens new window)

// 下面这些文本修饰方法并不会使字符串在文档中实际改变，而是采用合适的标签包裹和在标签上设置相关属性后返回处理后的html代码片段(有些标签或属性实际可能已废除)，需要自行插入到文档中才有效
str.big(); // 字体变大，返回示例：'<big>str</big>'
str.small(); // 字体变小
str.blod(); // 字体加粗
str.italics(); // 斜体
str.strike(); // 中划线
str.sub();  // 作为上标显示
str.sup();  // 作为下标显示
str.fontcolor("red"); // 字体颜色
str.fontsize(20); // 字体大小，不带单位，返回示例：'<font size="20">str</font>'
str.link("url"); // 添加链接，返回示例：'<a href="url">str</a>'

str1.localeCompare(str2); // 比较规则未详细说明，返回的结果类似数组的 sort 方法的排序规则，若 str1 比 str2 小则返回 -1，大则返回 1，相等则返回 0

# 查找

// 返回所查找字符第一次出现的位置，-1 表示未找到
str.indexOf(char[, start]); // char为需要查找的指定的字符，可选参数 start 表示开始查找的索引位置，默认是从位置 0 开始查找
// 同 indexOf，只是为倒序查找
str.lastIndexOf(char, start); // start 默认是 str.length-1，即最后一个字符
// 返回 index 位置的字符
str.charAt(index); // 当 index 为负数或大于等于字符串长度str.length时，将返回空字符串
// 返回 index 位置字符的数字编码
str.charCodeAt(index); // 当 index 为负数或大于等于字符串长度 str.length 时，将返回 NaN
// 静态方法，返回以 num 为数字编码的字符
String.fromCharCode(...num); // 多个num可用逗号隔开，注意前面是字符串原生对象 String
console.log(String.fromCharCode(67, 97, 116)); // Cat

# 截取

// 返回索引 start 与 end 之间的字符，不包括 end，此方法不会改变原数组，一般用slice方法较多
str.substring(start, end); // 若省略 end 则返回 start 及之后的所有字符，参数都为非负整数(若为负数或 NaN 则会被当做 0)，若 start 大于 end 则会自动调整换位
// 返回索引 start 开始共 n 个字符，此方法不会改变原数组
str.substr(start, n); // n 省略则返回 start 及之后所有字符，start 也省略则默认从 0 开始，start可为负值表示倒数(也可理解为其会自动加上字符长度 str.length)
// 同 substring 返回索引 start 与 end 之间的字符，不包括 end，但其参数可为负数，此方法不会改变原字符串，也可用于数组
str.slice(start, end); // 参数为负值时表示倒序查找(也可理解为负数会自动加上字符串长度)
// 返回拼接后的字符串，也可用于数组，不会改变原字符串和数组
str1.concat(...str2); // 拼接多个时参数用逗号隔开，可查看数组中的 concat() 方法说明

# 分割

// 以指定的符号 flag 或正则 regexp 为界将字符串 str 分割成字符串数组，并返回这个数组指定的 n 长度的数组段
// 若字符串中无指定的分割符号(或没传分割符)，则会将此字符串整体当作分割结果)，若省略 n 则返回整个分割后的数组
str.split('flag' | regexp[, n]);
//返回的数组中不包含分隔符，当分隔符为""空字符时可将字符串中每个字符都分隔开；当参数1是正则 regexp 时会忽略全局标识 g，另须注意正则(以匹配到的字符为分隔符)中分组捕获的文本也会被包含在返回的数组中

# 匹配查找替换

// 在运用正则进行字符串匹配时，正则的 regexp.lastIndex 属性的值会从 0 开始依次变为匹配到的字符串后下一个字符的下标(它表示每次开始进行匹配的位置)，不成功的匹配会重置regexp.lastIndex 为 0

// 将正则regexp用于匹配字符串str，匹配失败则返回 null，匹配成功则返回一个包含匹配到和正则捕获到的字符片段的数组（该数组还包含其他一些可用属性）
// 匹配成功返回的数组下标 0 单元格为匹配到的字符串(只包含一处)，下标 1 单元格返回分组1捕获的文本，以此类推，还有index属性为匹配位置的索引，input属性为最初输入的字符串等
//当正则带有全局g标识时，并不会匹配到所有值，每次值匹配一个，要匹配到所有则要循环调用该方法(每次regexp会记住当前匹配到的索引，下次将接着匹配后面的，而不是又从头开始)，所以此时须注意正则regexp和字符串str都要使用变量存储，否则在循环中每次的regexp和str都不一样(虽然字面值一样)，将无法完全匹配
regexp.exec(str);
console.log(/d/.exec('abcdefg')); // ['d', index: 3, input: 'abcdefg', groups: undefined]
// 用正则regexp测试字符串str中是否有匹配的字符，有则返回true，否则返回false
regexp.test(str); // 此方法正则若使用g全局标识则同上面exec类似，每次起始索引位置会接着上一次结束的位置，所以慎用g标识
// test方法全局g标识的影响
const regex = /foo/g;
regex.test('foofo'); // true，第一次执行，regex.lastIndex 为 0，即从 0 位置开始索引
regex.test('foofo'); // false，因上一次执行结束匹配到了内容导致 regex.lastIndex 变为 3，即此次是从 3 位置开始索引
// 匹配字符串中的指定字符 char 或正则 regexp，匹配到则返回包含该字符的一个数组(只包含首次匹配到的，该数组还具有一些其他可用属性)，未找到则返回 null
str.match(char | regexp); // 若参数是一个正则表达式 regexp，当正则不带 g 全局标识，则其返回的结果与 regexp.exec(str) 相同，而带有全局 g 标识时，则返回除捕获分组外的所有匹配的数组
console.log('abcdefg'.match('d')); // ['d', index: 3, input: 'abcdefg', groups: undefined]
// 返回指定字符或正则匹配到的字符串第一次出现的索引位置，未找到则返回 -1
str.search(char | regexp); // 参数为正则时可不要g全局标识，因为加了也会被忽略
// 用 char2 替换字符串 str 中匹配到的字符后并返回替换后的结果，该方法不会改变原字符串
// 参数2还可以是一个函数 fn，此时将对每个匹配到的文本片段依次调用此函数，该函数返回的字符串将被用作替换文本，传递给此函数的参数1为匹配到的文本片段，参数2为分组1捕获的文本，参数3为分组2捕获的文本，以此类推，然后最后还有两个参数分别为当前匹配的文本片段在整个字符串中的索引index、整个字符串str
// 注意默认只会替换其第一次匹配到的字符，包括正则不带 g 的情况，只有当正则带 g 全局标识时才会替换所有匹配到的位置
// 当参数1是正则，char2 为字符串且其中包含字符 $ 时，需注意此 $ 字符有特别含义：$$ 表示字符'$'、$& 表示整个匹配的文本、$n 表示第 n 个分组捕获的文本、$`表示匹配处前面的文本、$'表示匹配处后面的文本
str.replace(char1 | regexp, char2 | fn);
