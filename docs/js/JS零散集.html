# 值的比较

常见比较运算符：>、<、>=、<=、==、===、!=、!==

  • 当对不同类型的值进行比较时，它们会先被转化为数字（除===和!==）再进行比较，null被转化为0，注意undefined被转化为NaN，NaN与自身也不相等
  • 在非严格相等==下，null和undefined相等且各自不等于任何其他非自身的值
  • 字符串的比较，会按照“字母表”顺序逐字符地比较大小(实际是Unicode编码的大小)，后面的字母大于前面的，小写字母大于大写字母
  • 在使用>或<进行比较时，需要注意变量可能为null/undefined的情况，比较好的方法是单独检查变量是否等于null/undefined

与运算&&和或运算||返回的都是运算结果的原始值，而不是转换为布尔值，与运算&&优先级比或运算||高。

# 循环

++i/--i与i++/i--区别：前者是先自增/自减后的新结果参与所在代码块运算，而后者是先参与所在代码块运算后再进行自增/自减，其新结果得下次才能使用到，若单独使用而不参与循环代码块运算或在循环代码块之外使用时两者结果一样。

非表达式的语法结构不能与三元运算符? :一起使用，如break、continue语句。

break用于中止当前循环(不包括数组的forEach)、switch语句或label标记语句，并把程序控制流转到紧接着被中止语句后面的语句，注意其只能用于最小作用域是前面三种环境下，否则将抛出语法错误，比如数组的forEach，其参数是个函数，若最小代码块是该函数则break是不生效的，不会跳出该函数（可通过抛出错误来实现中断，但此时需要在外部catch抛出的错误，否则会影响后续程序执行，可查看数组Array中的示例）。

label标记语句所有浏览器都支持，包括低版本IE。

当要跳出多层循环时，可以使用label标记语句来跳出标记的代码块（当然我们也可以重构，将内层循环提取到一个新的函数中），如：

outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {

    let input = prompt(`坐标 (${i},${j})`, '');

    // 如果是空字符串或被取消，则中断并跳出这两个循环。
    if (!input) break outer; // 跳出指定的outer标记代码块，相当于直接跳出了两层循环，若不指定标记，则只是跳出当前的内层循环

    // 用得到的值做些事……
  }
}

continue则是跳过当前这一次循环体后面剩下的内容，而继续下一次循环，其也可以使用标记来实现多层循环中进入外层循环的下一次循环，而跳过内部循环中剩下的循环。

label标记语句语法就是在一条语句前面加一个自命名的标识符和冒号(也可单独一行)，标记可以是任意代码块，并非只能用于循环，但也只有break语句可以使用非循环标记，continue只能在循环中使用标记，如：

outer_block:
{
  inner_block:
  {
    console.log ('1');
    break outer_block;  // 跳出outer_block标记块
    console.log ('2');  // 跳过
  }
  console.log ('3');   // 跳过
}

# 函数

函数参数中的默认值可以是任何表达式，且其只会在缺少参数传入时才会被计算和分配，而非声明时就计算和分配。

function f1() {
  console.log('f1');
  return 'a';
}
function f2(a = f1()) { // 这里的 f1 在声明参数 a 时并不会立即执行和分配结果给 a
  console.log(a);
}
// 以下输出结果对比也说明默认值表达式只会在对应参数未传值时才会计算并分配
f2(1); // 1
f2(); // f1、a

要return的值必须跟return语句放在同一行，不能单独一行（内容中间可以换行，但不能return所在行后面无内容），否则return后面空内容会当做无返回值（因JS引擎会在return所在行末尾默认补上;分号）。

注释描述函数作用与用法：

/**
 * 返回 x 的 n 次幂的值。
 *
 * @param {number} x 要改变的值
 * @param {number} n 幂数，必须是一个自然数
 * @return {number} x 的 n 次幂的值
 */
function pow(x, n) {
  //...
}

# 对象

对象的属性名可以是任何字符串或Symbol，其他类型会被自动转换为字符串。若通过点符号.访问对象的key(即属性或方法)时，则要求key是有效的变量标识符，即意味着：不包含空格、不以数字开头、也不包含特殊字符(除$和_)，而中括号[]方式访问key则没有这些要求，且中括号中还可以结合表达式使用，能力更强。

in操作符可判断对象上是否存在某属性：prop in obj返回布尔值，但其会查找原型链，而不只是其自身属性。

for...in...遍历对象的所有非Symbol且可枚举的key(包括继承而来的)，可使用obj.hasOwnProperty(key)判断该key是否是该对象自身的。

for (let key in obj) {
  console.log(obj[key]);
}

Object.assign实现的对象拷贝是浅拷贝：

// 将后面(一个或多个)对象的属性拷贝给目标对象 targetObj，并返回 targetObj 的引用
Object.assign(targetObj, [srcObj1, srcObj2...]);

# 对象中的this/new

对象方法中的this指向点.之前的这个对象，也即调用该方法的对象。

this的值是在代码运行时计算出来的，它取决于调用代码上下文，而非定义时的上下文，箭头函数中没有自己的this，其this取决于外部父级作用域。

当一个函数被使用 new 操作符执行时，它按照以下步骤：

  1. 一个新的空对象被创建并分配给this（隐式创建）
  2. 函数体执行，通常它会修改this，为其添加新的属性
  3. 返回this的值（隐式返回）
function User(name) {
  this.name = name;
  this.isAdmin = false;
  if (!new.target) { // 函数内部可以通过 new.target 属性检查其是否被使用 new 进行调用，常规调用返回 undefined，new 调用会返回该函数自身
    // 若是常规调用，我们也可以让其实际按 new 调用，在封装插件时可以使暴露的调用语法更加灵活
    return new User(name);
  }
}
// 调用 new User(...) 时做的事就类似下面这样
function User(name) {
  // this = {}; (隐式创建)

  this.name = name;
  this.isAdmin = false;

  // return this; (隐式返回)
}

从技术上讲，任何函数（除了箭头函数，它没有自己的this）都可以用作构造器。即可以通过new来运行，它会执行上面的算法。“首字母大写”是一个共同的约定(并非强制)，以明确表示一个函数将被使用new来运行。

若我们有许多行用于创建单个复杂对象的代码，我们可以将它们封装在一个立即调用的构造函数中。立即调用的构造函数不能被再次调用，因为它不保存在任何地方，只是被创建和调用，该技巧旨在封装构建单个对象的代码，而无需将来重用。像下面这样：

// 创建一个函数并立即使用 new 调用它
let user = new function() {
  this.name = "John";
  this.isAdmin = false;
  // more code
};

# 构造器的 return

通常，构造器没有return语句。它们的任务是将所有必要的东西写入this，并自动转换为结果。但是，如果包含了return 语句，那么规则如下：

  • 如果return返回的是一个对象，则返回这个对象，而不是this；
  • 如果return返回的是一个原始类型，则忽略，默认转为返回this；

换句话说，带有对象的return返回该对象，在所有其他情况下(包括无return语句或return后无任何内容)都自动转为返回this。

如果构造函数没有参数，则实例化时也是可以省略后面的括号()的，虽然规范允许但这不是一个好的使用习惯：

let user = new  User; // 无参数
//等同于
let user = new User();

# 可选链 ?.

可选链?.用于其前面的值可能为undefined或null的情况(也即是所访问属性或方法是可选的)，符合情况时其会停止运算(语句剩下的调用部分不执行，即短路效应)并返回undefined，否则按照常规逻辑执行。

使用可选链的变量一定要是声明过的或者是函数的参数，否则会报错。也不要过渡使用可选链，而应该只用在一些东西允许不存在的地方，也即是所访问属性允许可选的情况。

可选链?.不是一个运算符，而是一个特殊的语法结构，故其还可以与函数和方括号一起使用。

可以使用可选链?.来安全地读取或删除，但?.不能用在写入中，也即不能用在赋值语句的左侧。

因数字、字符串、布尔值有包装对象，故也可像对象调用一些属性或方法，但数字只能通过变量或对象属性存储方式调用，或者用括号()包裹，否则会报错：let n=7; n.a不报错、7.a报错、(7).a不报错。

console.log(obj?.prop); // 报错，因为使用可选链的变量未声明

const obj = { prop1: 1 };
console.log(obj?.prop); // undefined，这里与直接 obj.prop 差不多
console.log(obj.prop?.ziprop); // undefined，这里就体现出可选链的作用了
console.log(obj.prop1?.ziprop); // undefined，因这里 obj.prop1 不为 undefined 或 null，则 obj.prop1?.ziprop === obj.prop1.ziprop
console.log(obj.prop1?.z?.zz); // undefined，表示 prop1 可选，prop1 下的 z 也是可选的
console.log(obj.prop?.z.zz); // undefined，表示 prop 可选，prop 下的 z 也是可选的

let html = document.querySelector('.el')?.innerHTML;  // 如果没有找到符合的元素，则会返回 undefined，若不使用可选链则元素不存在时访问 innerHTML 就会报错

const obj2 = null;
obj2?.foo();  // obj2 为 null，故这里后面的 foo() 部分不会执行

// ?. 并非一个运算符，而是一个特殊的语法结构，故其后可以紧跟函数调用`()`或属性访问`[]`
obj.foo?.();  // 不会报错，因为 obj.foo 为 undefined，则后面的`()`不会继续执行，也就不会形成方法调用
obj.prop?.['child-prop'];  // 不会报错，若 obj.foo 不为 undefined 或 null，则结果等于 obj.foo['child-prop']，否则为 undefined
// 与 delete 一起使用
delete obj2?.foo;
obj2?.prop = 7;  // 报错，可选链不能用于赋值操作左侧

总结可选链?.的三种使用形式：

  1. obj?.prop，最常见的形式；
  2. obj?.[prop]，对象的中括号访问语法，有效则返回obj[prop]，否则返回undefined；
  3. obj.method?.()，若有效则调用obj.method()，否则返回undefined；

# Symbol

只有两种原始数据类型可用于对象的属性或方法名中，一种就是一直都存在的字符串(其他的会自动转为字符串)，另一种就是新增的表示唯一标识符的Symbol

// 创建 Symbol
let sym = Symbol();  // 可以不传任何参数
let sym1 = Symbol('id');  // 可以传入一个描述该Symbol的名字，调试代码时较有用
let sym2 = Symbol('id');  // 即使传入同样的Symbol名，得到的值也是不一样的
console.log(sym1 == sym2);  // false

// 传入 Symbol 的描述信息可以通过其 description 属性获取到
console.log(sym1.description);  // id

Symbol 不会被自动隐式转换为字符串，JS中大多数值都支持字符串的隐式转换，如我们可以alert任何值都生效(只是有的显示形式不一样，就是被进行了隐式字符串转换)，但 Symbol 就比较特殊，它若被隐式进行自动转换就会报错，如alert(Symbol('id'))就会报错 Uncaught TypeError: Cannot convert a Symbol value to a string

若实在要显示 Symbol 的字符串形式，可以显式调用其toString()方法：

alert(sym1.toString()); // 弹出 Symbol(id)

# 使用Symbol创建隐藏属性

用 Symbol 创建对象 obj 的属性可以使得该属性对外是隐藏的，因为只有在编写代码时通过变量 key 保存了该 Symbol 作为的键名，才能在后续通过这个变量 key 访问该 Symbol 对应的键值，显然我们对外不暴露这个 key 变量时，外部也就不知道如何访问这个 Symbol 键名对应的值，因为 Symbol 也只有通过存储它的变量来使用才有意义(保证每次用的 Symbol 是同一个值)，否则每次直接调用实际都会是一个新的唯一值。
