# async/await

async函数使得异步操作更加方便(该关键字表明函数体内有异步操作)，其返回一个Promise对象(注意其并不是使用return返回Promise对象，而是这样定义异步函数时就自动表示有这样的返回，而其函数体内的显式return的值会作为参数传给then()回调)，可以使用then()方法添加回调函数。当函数执行的时候，一旦遇到await(其后应是一个异步操作)就会先返回，等到异步操作完成，再接着执行函数体内后面的语句。

async函数也可以使用在多种情况下(也就是在原普通函数声明方式的前面加个async关键字，所以普通函数所有的使用场景也都适用于async函数)，如：直接async函数声明、函数表达式(赋给一个变量)、作为对象的方法、作为Class的方法、箭头函数形式声明。

async function getCardInfo() {
  const sdk = new WriteCardSDK("WebSocket");
  // 注意：await后面通常情况下应该是一个Promise对象，await会返回其后异步操作的结果（注意是结果，不是异步操作对象本身）
  const version = await sdk.getSdkVersion(); // 执行到这里该getCardInfo函数就会返回一个Promise对象，然后继续执行getCardInfo函数外部后面的代码
  const res = await sdk.getCardSN(version); // 这里的await异步操作也会等上面的await操作完成后在再行，可看作是前面异步操作之后的回调代码（虽然代码结构看着像同步）
  console.log(res);	// 这里的代码会继续等上面最近的await异步操作执行完之后再执行，又可以看作是前面第二个异步操作之后的回调代码
  return res; // 该return值会作为getCardInfo函数的then回调的参数传入
  // 只有`async`函数内部的所有异步操作执行完(或出现错误)，致使其状态改变，最后才能触发其外部指定的`then`或`catch`回调
}

getCardInfo().then((res) => {
  console.log('这里的`res`参数是上面`getCardInfo`方法中最后显式return出来的结果');
});

// 因async函数返回一个Promise对象，所以async函数也可以作为await命令后的参数
async function printCardInfo() {
  await getCardInfo(); // 这里getCardInfo是上面的async函数
  console.log('getCardInfo操作完成了，可以执行当前回调了');
}

async函数内部抛出错误(主动抛出或是其内任一个await操作出现reject状态)，会导致返回的Promise对象立即变为reject状态而终止执行其内部抛出错误位置后面的代码，抛出的错误对象会被catch方法回调函数接收到(此时即使没有显式return错误信息也是会被catch到的)。

async function foo() {
  throw new Error('出错了');
}

foo().then(
  v => console.log('resolve', v),
  e => console.log('reject', e)
)
// 或者写成
foo().then(v => console.log('resolve', v))
     .catch(e => console.log('reject', e));
//输出结果：reject Error: 出错了

有时若我们希望前面的异步操作出错时不要中断后面的异步操作，此时可以将前面需要做容错处理的await异步操作放在try...catch结构中，这样不管该异步操作是否成功，其后面的await异步操作都不受影响而会继续执行。或者将前面需要做容错处理的await后的Promise加上catch回调以捕获可能的错误信息。

async function foo2() {
  await Promise.reject('出错了');
  await Promise.resolve('hello world'); // 因前一个await变为reject，所以这里不会再执行
}
// 改写为
async function foo2() {
  try {
    await Promise.reject('出错了'); // try...catch 中可以同时放置多个await异步操作
  } catch(e) {
  }
  await Promise.resolve('第二个await成功');
}
// 或改写为
async function foo2() {
  await Promise.reject('出错了').catch(e => console.log(e)); // catch 回调只能对每个 Promise 单独设置
  await Promise.resolve('hello world');
}

若async函数中存在多个await命令，但每个await命令后的异步操作不存在继发关系时(即没有前后顺序依赖关系)，则最好让这些异步操作同时触发，以减少总体的等待耗时。

async function foo() {
  let foo = await getFoo();
  let bar = await getBar(); // 这里的异步操作需要等上面的异步操作完成才开始执行(总的耗时为t1+t2)，但是这里两个异步操作实际并不存在顺序依赖关系，若同时执行的话耗时只是max(t1,t2)
}
// 改写为
async function foo() {
  let [foo, bar] = await Promise.all([getFoo(), getBar()]); // 使用`Promise.all()`方法将多个Promise组合成一个Promise
}
// 或改写为
async function foo() {
  let fooPromise = getFoo();
  let barPromise = getBar();  // 该写法两个异步操作是同时执行的
  let foo = await fooPromise; // 因所有异步操作已在前面开始执行，后面的`await`只是在等待结果，就不会中断第二个异步操作
  let bar = await barPromise;
}

async函数返回的Promise对象，必须等到内部所有await命令后面的Promise对象执行完，才会发生状态改变，除非提前遇到return语句或者抛出错误，即只有async函数内部的所有异步操作执行完，才会执行then方法指定的回调函数(注意这里说的是执行，并不是async函数本身此时才返回Prommise，返回Prommise是在其函数体内部遇到await命令时就返回了)。

正常情况下，await命令后面是一个Promise对象，并会返回该对象的结果。如果不是Promise对象，就直接返回对应的值。另一种情况是，await命令后面是一个定义了then方法的对象(自定义的，并非Promise，但又是按Promise中then使用方式定义的)，那么await会将其等同于Promise对象对待。

async function foo() {
  return await 123; // 等同于：return 123
}

// 使用`async/await`和`setTimeout`定义一个睡眠函数(这里Sleep只是举例，可用`Promise`简化)
class Sleep {
  constructor(timeout) {
    this.timeout = timeout;
  }
  then(resolve, reject) { // 自定义的`then`方法(类似`Promise`中)
    const startTime = Date.now();
    setTimeout(() => resolve(Date.now() - startTime), this.timeout);
  }
}
(async () => {
  const sleepTime = await new Sleep(1000); // 此时`await`就会将`Sleep`当做`Promise`对待，会等`Sleep`中的异步操作完成后再执行下面的内容
  // 此睡眠函数实际与直接使用`setTimeout`本质上是一样的，只是说`setTimeout`需要将后续代码放在回调中，而该`async/await`改写的方式同步代码结构上看着更清晰
  console.log(sleepTime); // 输出结果：1000
})();
