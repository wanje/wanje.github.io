# Vue工程创建及配置

  • 使用vue-cli脚手架创建vue项目

# 使用vue-cli脚手架创建vue项目

命令行中两个杠--的表示指令，一个杠-的表示参数。

参数-g表示全局安装，安装后以后都能用，不必每个项目再单独安装，可以通过vue -V命令判断是否安装vue-cli脚手架。

# 全局安装官方脚手架
 npm install @vue/cli -g # 包名已由1.x/2.x的`vue-cli`改为`@vue/cli`
 yarn global add @vue/cli # yarn

 # 若已安装旧版 vue-cli，则应先卸载
 npm uninstall vue-cli -g # npm
 yarn global remove vue-cli # yarn

 # 安装脚手架后可通过帮助命令查看脚手架都有哪些操作命令
 vue -h

脚手架安装后就可以根据提供的模板初始化一个vue项目了(也可以不用模板而直接创建一个vue项目)，webpack是模板名(可使用官方提供的其他模板，也可以自定义模板custom)，projectName是创建的该vue项目的名字。

# 创建一个基础工程（cli3+已内置webpack的基本配置）
 vue create projectName

# 使用 cli2 的模板初始化一个vue工程（cli3+与cli2采用相同的vue命令，但这里只是拉取2.x的模板，cli3+的已改为内置，故cli3直接采用上面create的方式，或使用vue ui）
 vue init webpack projectName # 初始化一个基于webpack模板的vue工程

 # cli3+ 版本可以直接通过GUI图形化界面进行创建
 vue ui

 # Mac下若权限不够则需要使用管理者权限执行命令，命令前加`sudo`

执行上面初始化命令时会提示选择或输入该项目的相关信息及是否使用一些代码检测等辅助工具，操作完成后会提示后面你需要做的三步(即下面3步)。

进入刚刚创建的项目文件夹，项目文件夹是自动生成在上面模板初始化命令所在执行位置的，所以想在哪里创建项目，就要在哪个文件夹下执行步骤2的命令。

 cd projectName # 进入创建的工程目录下

下面的命令是安装默认的一些依赖包，不然工程运行不起来，这里默认安装的是开发环境依赖，若只安装生产环境依赖则后面要加指令--production，对应则安装dependencies下的依赖。

 npm install # 安装初始化工程中已添加的依赖

对于cli3+，若需要自定义一些额外的配置，可以在项目根目录下(与package.json同级)创建一个vue.config.js文件进行相关配置。

// vue.config.js
module.exports = {
  // 选项...
}

下面的命令是以开发模式运行该vue项目，npm run build则是打包压缩(生产模式)，其实执行的都是package.json文件中"scripts"配置下的命令(最终指向一些nodejs)。

 npm run dev # 若package.json中scripts下配置了start，则也可以使用npm start

至此就可以根据情况进行项目开发了。
