    # 标题

    ## 1-6级标题
h1-h6依次是以井号“#”的个数表示，几级标题就使用几个“#”符号，且后面`留个空格`再写标题名，每个标题与上一行之间应该隔开至少一行

分级标题一级“=”
===========
分级标题用“=”号或“-”号，“=”为一级标题，“-”为二级标题，最少可使用一个符号标识，也可以多个用以分开，`分隔线上面一行即为标题`，效果同“#”

分级标题二级“-”
----------
分级标题二级用“-”符号，效果同“##”，注意有些markdown语法中“-”是用作无序列表的标识符的（同下面的“*”）

普通段落文本不需要特殊标记

    # 引用

    使用`>`大于符号（右箭头）表示引用，引用内容左侧边会出现一个较粗的边框，引用嵌套可用 “>” 符号的个数表示。
如：`>>>` 表示是嵌套的第三层

    >第一层引用

    >> 嵌套第二层引用

    >>> 嵌套第三层引用

    # 代码标记

    # 行内代码标记，前后使用一个反引号包裹：` 
单反引号在ES6中用作字符串模板，这里是一个 ` <br/> ` 标签，标记出的内容将高亮显示，可在反引号前后添加一个空格。

# 代码块标记，前后使用三个反引号包裹：```
在代码块顶部三个反引号后指明语言类型(可使用简写名称)可使得代码按相应格式高亮显示，不指名则就默认为一般文本。
代码块将独占整行，为方便查看原始markdown文件内容，可以与上面隔开一行。

    <!-- html -->
<ul class="test">   
    <li>item1</li>
    <li>item2</li>
    <li>item3</li>
</ul>

    // javascript
let num = 0;
for (let i = 0; i < 5; i++) {
    num+=i;
}
console.log(num);

 display: inline;

    # 插入链接

    语法：`[显示的、需要添加链接的文本](链接地址)`
添加跳转方式{:target="_blank"}(有兼容问题)：`[显示文本](链接地址){:target="_blank"}`，vuepress中外链默认就在新窗口打开

    参考文章：https://www.jianshu.com/p/b03a8d7b1719 (opens new window)

    # 插入图片

    语法：`![图片alt](图片地址 '图片title')`
带链接的图片：`[![图片alt](图片地址 '图片title')](链接地址)`

这里vuepress中也可以直接使用`img`元素，方便进行一些样式设置：<img src="~@/images/knowledge/css/zIndex-order.jpg" width="500" />

    # 列表

    # 有序列表

    语法：`1. 数字标号加小圆点“.”，不能用顿号1、的形式，内容与标号间空格隔开，否则列表不会按序换行显示`
1. 列表嵌套
	1. 列表嵌套要与上一级有缩进，否则不会展示层级
	2. 根据编辑器不同，嵌套后实际显示的子列表标号有所不同
    1. 有序1
    2. 有序2
      1. 嵌套2-1
      2. 嵌套2-2

    # 无序列表

    语法：`* 内容`（注意有些markdown语法中是将“-”用作无序列表标识符的）
* 使用星号“ * ”代表列表，内容与前面星号间同样空格隔开，否则不会换行
	* 列表嵌套要与上一级有缩进，否则不会展示层级
	* 有序、无序列表间可互相嵌套
    • 无序1
    • 无序2
      • 嵌套2-1
      • 嵌套2-2

    # 列表嵌套

    1. 列表可多层嵌套
    2. 有序、无序列表间可互相嵌套
      • 列表嵌套要与上一级有缩进，否则不会展示层级
      • 根据编辑器不同，嵌套后实际显示的子列表标号有所不同
        1. 列表嵌套要与上一级有缩进，否则不会展示层级
        2. 根据编辑器不同，嵌套后实际显示的子列表标号有所不同

    # 任务列表

    表示选中与否，类似行首带复选框，可用于表示任务完成与否的清单，不同编辑器下效果可能不同(有些可能不支持)
语法：`- [x] 选项`
通过中括号中有无 x 来表示当前项为选中状态(或已完成)还是未选中状态，注意前面的中划线“ - ”和中括号“[ ]”和后面的选项文字之间都要用空格隔开
- [x] 选项一，有 x 表示选中状态
- [ ] 选项二，无 x 则表示未选中状态
    • [x] 选项一，中括号中有x表示当前项为选中状态(或已完成)
    • [ ] 选项二，中括号中为空格不包含x则表示未选中状态

    # 表格

    语法：冒号加短横线组合 `:-、-:、:-:` 且之间不能有空格，短横线可以连续多个，冒号‘:’相对短横线‘-’的位置表示单元格内容对齐方向
冒号‘:’与表示单元格分割线的竖线“|”间也不能有空格、第一行为标题，第二行设置对齐方式，第三行开始为表内容

**也可直接使用HTML中的table表格，要合并单元格时就只能使用HTML了**

|  表头一  |     表头二     |    表头三   |
|:-------:|:------------- | ----------:|
| 居中内容 |   左对齐内容   |  右对齐内容  |

使用一个短横线更简洁简写：
a|b|c  
:-:|:-|-:
居中内容|左对齐内容|右对齐内容
    表头一 表头二 表头三
    居中内容 左对齐内容 右对齐内容
    表头一 表头二 表头三
    居中内容 左对齐内容 右对齐内容

    # 文件树形结构

    markdown本身并没有这样的语法，而是采用其他工具生成之后直接复制过来的，比如Mac下可以安装tree工具：

    brew install tree # 安装tree工具

# tree 部分常用命令
tree -a # 显示所有
tree -d # 只显示文件夹
tree -L <level> # 显示项目的层级，level表示层级数，比如想要显示项目三层结构，可以用tree -l 3；
tree -I pattern # 用于过滤不想要显示的文档或者文档夹。比如你想要过滤项目中的 node_modules 文档夹，可以使用 tree -I “node_modules”；
tree > tree.md # 将项目结构输出到 tree.md 这个文档

    将树形结构放在代码块或者<pre>标签中就可以保持结构不变

    ├─┬ registry-auth-token@4.2.1
│ └─┬ rc@1.2.8
│   ├── deep-extend@0.6.0
│   ├── ini@1.3.8
│   ├── minimist@1.2.5 deduped
│   └── strip-json-comments@2.0.1

    # 斜体

    • 前后使用一个星号*包裹 *斜体xieti* ：斜体xieti
    • 前后使用一个下划线_包裹 _斜体xieti_ ：斜体xieti
    • 直接用 <i> 标签 <i>斜体xieti</i> ：斜体xieti

    # 加粗

    • 前后都使用两个星号**包裹 **粗体** ：粗体
    • 直接用 <b> 标签 <b>粗体</b> ：粗体

    # 加粗+斜体

    即上面两种的结合，用一种符号包裹后再用另一种符号包裹即可

    • 即使用三个星号包裹（常用） ***加粗+斜体xieti*** ：加粗+斜体xieti
    • 粗体包裹斜体 **_加粗+斜体xieti_** ：加粗+斜体xieti
    • 斜体包裹粗体 _**加粗+斜体xieti**_ ：加粗+斜体xieti
